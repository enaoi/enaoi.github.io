# 你不知道的JAVASCRIPT

# scope

# 第一章：作用域

一套用来存储和访问变量的规则，称为作用域

当一个块或函数嵌套在另一个块或者函数中时，就发生了作用域嵌套。在当前作用域中无法找到某个变量时，引擎就会在外层嵌套的作用域中继续查找，直到找到该变量，或抵达最外层的作用域为止。

# 第二章：词法作用域

作用域工作模型常见有2种：

  • 动态作用域：Bash/Perl
  • 词法作用域：JavaScript

定义在词法阶段的作用域叫做词法作用域

JavaScript 存在2种欺骗词法作用域的机制：

  • eval：动态执行字符串语句
  • with：改变当前块的作用域

# 第四章：提升

声明在编译阶段进行，赋值留在原地等待执行阶段，看起来声明被提升了。

每个作用域都会进行提升操作。

具名的函数表达式名称标识符在赋值之前无法在所在函数作用域中使用。

例子 
foo() //TypeError
bar() // ReferenceError
var foo = function bar() {}

提升后理解为：

var foo
foo()
bar()

var foo = function() {
  var bar = // ...self...
}

函数优先：函数声明和变量声明都会被提升，但函数会首先被提升，然后才是变量。

# 第五章：作用域闭包

当函数可以记住并访问所在的词法作用域时，就产生了闭包。

闭包形成的解析 
function foo(){
  var a = 2;
  function bar(){
    console.log(a)
  }
  return bar
}

var baz = foo()

baz() // 2
  • 在函数foo执行后，一般会期待引擎的垃圾回收机制将函数foo整个内部作用域都被销毁，来释放内存。
  • 而闭包保持着对这个函数内部作用域的引用，而这个引用就叫做闭包。

无论已何种手段，将内部函数传递到所在词法作用域以外的地方，它都会持有对原始定义作用域的引用。无论在何处执行这个函数都会使用闭包。

# 模块

模块模式的2个必要条件：

  1. 必须有外部封闭函数，该函数至少调用一次。
  2. 封闭函数必须返回至少一个内部函数。这样内部函数才能在私有作用域中形成闭包，并且可以访问或修改私有的状态。

参见单例模式
