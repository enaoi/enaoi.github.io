vue add vue-next

# 动机和目的

目前的缺陷：

  1. 功能增长，复杂组件越来越庞大
  2. 缺少低成本复用和提取组件之间的逻辑
  3. 对于TypeScript的支持有限

相对于 mixins、高阶组件或者是插槽实现的模式的缺点：

  • 渲染上下文暴露的 property 来源不清晰，看不出属性是哪个 mixin 注入的。
  • 命名空间冲突
  • 高阶组件和无渲染组件需要额外的有状态组件实例，性能有所消耗

# 学习

# setup

调用时机: 创建组件实例 => 初始化props => 调用setup函数 => beforeCreate钩子。

使用:

  1. 返回一个对象，该对象的属性自动合并到组件模板的上下文。
  2. 返回一个渲染函数。
import { h, ref, reactive } from 'vue'
export default {
  setup () {
    const count = ref(3)
    const object = reactive({ foo: 'bar' })
    // 返回一个对象
    // return { count, object }
    // 返回一个渲染函数
    return () => h('div’, [count.value, object.foo])
  }
}
</script>

# 响应式API

# reactive

const obj = reactive({count: 0}): 接受一个普通对象后返回该对象的响应式代理。等同于Vue.observable()。该响应式转换是“deep”的，会影响对象内所有嵌套的属性。基于ES2015的proxy实现，返回的代理对象不等于原始对象。

# ref

cont count = ref(0) 接受一个参数值并返回一个响应式且可改变的ref对象。ref对象拥有一个指向内部的单一属性.value

  • 如果传入ref的是一个对象，将会调用reactive方法进行深层响应转化。
  • 作为渲染上下文的属性返回，setup函数会自动解套，无需书写.value
  • 只有嵌套在reactive的Object中时，ref才会解套。Array 和 Map 不会解套

JavaScript 中基础类型是值传递而非引用传递。一个响应式的值一旦作为 property 被赋值或从一个函数返回，而失去了响应性之后，也就失去了用途。为了确保始终可以读取到最新的计算结果，我们需要将这个值上包裹到一个对象中再返回。

// 伪代码
const ref = {
    value: null,
}

# computed

  • 传入一个get函数，返回一个默认不可修改的ref对象
  • 传入get和set函数，返回一个可手动修改的计算状态
const count = reactive({ count: 0 })

const readonly = computed(() => counte.value + 1)

const writable = computed({
  get: () => {
    count.value + 1
  },
  set: (value) => {
    count.value = value - 1
  }
})

# copy

传入一个对象或者是ref，返回一个原始对象的只读代理。只读属性是深层的，任何内部嵌套属性都是只读。修改会导致warning。

const original = reactive({ count: 0 })
const copy = readonly(original)

// warning
copy.count++

# watchEffect

传入一个立即执行函数，响应式追踪依赖，在依赖改变时，重新调用该函数

停止侦听:

  • 在setup()函数或生命周期钩子被调用时，侦听器会被链接到该组件的生命周期，并在组件卸载时自动停止。
  • const stop = watchEffect(() => {})显式调用返回值以停止侦听。stop()

WARNING

清除副作用，以及副作用刷新的时机待完善和理解

# watch

watch 等同于 this.$watch。侦听特定的数据源，并且在回调函数中执行副作用。默认情况是懒执行的，仅在数据源发生变化时才会执行。

// 1. 侦听一个 getter
const state = reactive({ count: 0 })
watch(
  () => state.count,
  (count, preCount) => {
    // 副作用...
  }
)
// 2. 侦听一个 ref
const count = ref(0)
watch(count, (count, preCount) => {
  // 副作用...
})
// 3. 侦听多个数据源
watch([aRef, bRef], ([a, b], [prevA, prevB]) => {
  // ...
})

# 生命周期钩子函数

这些钩子函数只能在 setup 函数内使用

  • beforeCreate -> 使用 setup()
  • created -> 使用 setup()
  • beforeMount -> onBeforeMount
  • mounted -> onMounted
  • beforeUpdate -> onBeforeUpdate
  • updated -> onUpdated
  • beforeDestroy -> onBeforeUnmount
  • destroyed -> onUnmounted
  • errorCaptured -> onErrorCaptured

新增钩子函数:

  • onRenderTacked
  • onRenderTriggered

# 依赖注入

provide 和 inject ，都只能在 setup 中调用。如果inject未提供默认值，上下文也未找到，则返回undefined。

可以使用 ref 来保证 provide 和 inject 之间的响应。

const ThemeSymbol = Symbol('theme')
const ThemeRef = ref('dark')

// parent.vue
setup () {
  provide(ThemeSymbol, ThemeRef)
}

// children.vue
setup () {
  // ligth 是默认值
  let theme = inject(ThemeSymbol, ref('light'))
  watchEffect(() => {
    console.log(theme.value)
  })
}

# 模板Refs-DOM引用

对模板或组件实例的引用，一样在 setup 函数中声明一个 ref 并返回它。

  • Virtual DOM patch 算法中，如果一个VNode的 ref 对应一个渲染上下文中的ref，则该元素或者组件实例就会分配给该 ref。
  • 这是在 Virtual DOM 的 mount / patch 过程中执行的，因此模板 ref 仅在渲染初始化后才能访问。
  • v-for 中需要使用函数型 ref

# 高级响应系统API

# customRef

用于自定义一个 ref，可以显式地控制依赖追踪和触发响应。

<template>
  <div class="home" ref="home">
    <input v-model="text" />
  </div>
</template>

<script>
import { customRef } from 'vue'

function useDebuouncedRef (value, delay = 200) {
  let timeout
  return customRef((track, trigger) => {
    return {
      get () {
        track()
        return value
      },
      set (newValue) {
        clearTimeout(timeout)
        timeout = setTimeout(() => {
          value = newValue
          trigger()
        }, delay)
      }
    }
  })
}

export default {
  setup () {
    return {
      text: useDebuouncedRef('hello')
    }
  }
}
</script>

# markRaw

显式标记一个对象为“永远不会转为响应式代理”，函数返回这个对象本身。

如果被 markRaw 标记了，即使在响应式对象中作属性，也依然不是响应式的。

# shallowReactive、shallowReadonly

只为某个对象的私有（第一层）属性创建浅层的响应式代理。

# shallowRef

创建一个 ref ，将会追踪它的 .value 更改操作，但是并不会对变更后的 .value 做响应式代理转换（即变更不会调用 reactive）

# toRaw

返回由 reactive 或 readonly 方法转换成响应式代理的普通对象。这是一个还原方法，可用于临时读取，访问不会被代理/跟踪，写入时也不会触发更改。

# Ref vs Rreactive

Q: 什么时候使用 ref 什么时候使用 reactive？

ref 和 reactive 更类似于一种风格的不同。ref 使用风格1，reactive使用风格2。

// 风格 1: 将变量分离
let x = 0
let y = 0

function updatePosition(e) {
  x = e.pageX
  y = e.pageY
}

// --- 与下面的相比较 ---

// 风格 2: 单个对象
const pos = {
  x: 0,
  y: 0,
}

function updatePosition(e) {
  pos.x = e.pageX
  pos.y = e.pageY
}
