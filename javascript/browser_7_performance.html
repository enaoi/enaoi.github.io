# performance

传统计算一个页面加载时间：页面第一个 script 被执行至整个页面加载完成的时间。

<!DOCTYPE html>
<html>
<head>
  <script>
    var start = new Date().getTime();`
    function onLoad() {
      var now = new Date().getTime();
      var latency = now - start;
      console.log(`page loading time: ${latency} ms`)
    }
  </script>
</head>
<body onload="onLoad()">
    <!-- Main page body goes from here. -->
</body>
</html>

# DNS查询

  • domainLookupStart
  • domainLookupEnd 当用户请求一个 URL 时， DNS（Domain Name System）需要将 domain 转化为对应的 IP 地址。DNS的查询时间受到各种因素的影响，尤其是 DMS 缓存。Navigation 和 Resource Timing 都包含这类属性。
const dnsTime = navigation.domainLookupEnd - navigation.domainLookupStart
1

但 domainLookupEnd 和 domainLookupStart 可能都是 0。比如一个资源是第三方提供的，且第三方主机没有设置 Timing-Allow-Origin。

# 连接协商

  • connectStart：客户端向服务端打开连接
  • secureConnectionStart：TLS协商，可能值为 0
  • connectEnd：协商结束 TLS 协商的结束时间和 connectEnd 相同
// Quantifying total connection time
var pageNav = performance.getEntriesByType("navigation")[0];
var connectionTime = pageNav.connectEnd - pageNav.connectStart;
var tlsTime = 0; // <-- Assume 0 by default

// Did any TLS stuff happen?
if (pageNav.secureConnectionStart > 0) {
  // Awesome! Calculate it!  
  tlsTime = pageNav.connectEnd - pageNav.secureConnectionStart;
}

# 请求和响应

  • fetchStart：当浏览器开始请求资源时标记，浏览器开始检查缓存（HTTP和service worker 缓存）
  • workerStart：当从 service worker 接收到 fetch 事件返回。
  • requestStart
  • responseStart：响应的第一个字节到达时间
  • responseEnd：响应的最后一个字节到达时间
const fetchTime = navigation.responseEnd - navigation.fetchStart;

let workerTime = 0;
if ( navigation.workerStart > 0 ) {
  workerTime = navigation.responseEnd - navigation.workerStart;
}

// 只有请求时间 (抛出 unload, redirects, DNS, and connection 时间)
const requestTime = navigation.responseStart - navigation.requestStart;
// 响应时间 (download)
const responseTime = navigation.responseEnd - navigation.responseStart;
// Request + response
const requestResponseTime = navigation.responseEnd - navigation.requestStart;

# 文件和资源大小

  • transferSize：资源的总体大小，包括 HTTP 头部。但有的值为 0。
  • encodedBodySize：压缩后的文件大小，不包括 HTTP 头部。
  • decodedBodySize：解压缩之后的文件大小，不包括 HTTP 头部。
// resource
const headerSize = resource.transferSize - resource.encodedBodySize;

# performance.navigation

用户是如何到达这个页面的?

参数 参考值 说明
redirectCount 0 页面通过几次重定向跳转来
type 1 通过何种方式进入页面，枚举类型[ 0, 1, 2, 255 ]

其中type为枚举类型

数值 NavigationType 说明
0 "navigate" TYPE_NAVIGATE：当前页面是通过点击链接，书签和表单提交，或者脚本操作，或者在url中直接输入地址
1 "reload" TYPE_RELOAD：点击刷新页面按钮或者通过Location.reload()方法显示的页面
2 "back_forward" TYPE_BACK_FORWARD: 页面通过历史记录和前进后退访问时
255 "prerender" TYPE_RESERVED：其他

# performance.timing

导航和页面加载事件的数据

performanceTiming 接口

# performance.now()

返回相对 performance.timing.navigationStart 的时间， 微秒（百万分之一秒）为单位，更加精准。 相对于 Date.now() 的优点：

  • 自身不会阻塞和影响页面
  • Date 是不精确的
  • 不会受到系统程序执行阻塞的影响
  • 不受系统时间的影响，认为修改或程序调整系统时间对它无影响
  • Date无法计算页面加载前的延迟时间

# 跨域和 Timing-Allow-Origin

如果Timing-Allow-Origin 头部未设置的跨域请求，performance timing是无法统计到的。

# HTTP 持久连接

http1 头部设置 Connection: Keep-Alive 。这些都会影响 timing。
