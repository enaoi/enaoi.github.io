# 组件组合

  • 特殊的 prop children：让组件通过 JSX 嵌套，将任意组件作为子组件传递。
  • “接口 slots”：将所需 JSX 内容传入 props，并使用相应的 prop
// 调用
<FancyComponent title={<h1>this is title</h1>}>
    <CustomComment/>
</FancyComponent>

// 内部实现
<div className="fancy-component">
    <div className="header">{props.left}</div>
    <div className="content">
        {props.children}
    </div>
</div>

# Context

参考文档：context 源码分析

context 的使用场景：

  • 在构建某些独立的复杂组件时，组件内的全局状态管理，避免了 props 逐层传递。
  • 简单系统的全局变量，如用户信息，主题和首选语言等，不必引入第三方状态管理库。

不一定使用 context 的场景：

  • 多层嵌套后，只有一个子组件渲染需要传递的 props。可以直接传递这个子组件本身。
  • 传递多个子组件时，可以封装多个单独的“接口（slots）”
  • 如果子组件需要在渲染前和父组件进行一些交流，可以进一步使用 render props。

🍒  传统构造

// 创建，传入默认值
const ThemeContext = React.createContext("light")

//组件调用,我们将 “dark” 作为当前的值传递下去。
<ThemeContext.Provider value="dark">
    <YourComponent/>
</ThemeContext.Provider>

// 消费组件

# Render Props

React 组件之间使用一个值为函数的 prop 共享代码的简单技术

// 调用
<FancyComponent render={fancyData => (<FancyData value={fancyData}/>)} />

// 使用 `render`prop 动态决定要渲染的内容，而不是一个组件渲染结果的静态表示
<div className="fancy-component">
    {this.props.render(this.state.fancyData)}
</div>
  • render props 不一定需要命名为 render，任何函数形式的 prop 都可以
  • render prop 会抵消使用 pure component 带来的优势，因为浅比较 props 得到的都是不相等。解决方案如下：
class MouseTracker extends React.Component {
  // 定义为实例方法，`this.renderTheCat`始终
  // 当我们在渲染中使用它时，它指的是相同的函数
  renderTheCat(mouse) {
    return <Cat mouse={mouse} />;
  }

  render() {
    return (
      <div>
        <h1>Move the mouse around!</h1>
        <Mouse render={this.renderTheCat} />
      </div>
    );
  }
}

# Refs

🍒  Refs & DOM

适合使用 refs 的情况：

  • 管理焦点，文本选择或媒体播放。
  • 触发强制动画。
  • 集成第三方 DOM 库。

创建：let myRef = React.createRef(), 作为 ref 属性的初始值。

访问：const node = this.myRef.current。当 ref 属性作用在不同类型元素上，接收类型不同：

  • HTML 上的 ref.current 接收的是底层 DOM 元素
  • 自定义 class 组件上的 ref.current 接收组件的挂载实例
  • 不能在函数组件上使用 ref 属性，因为它们没有实例。解决方案：
    • 将函数组件改写为 class 组件。
    • 使用 Refs 转发函数组件的 ref 到子组件或 DOM 上。

🍒  回调 Refs

  • 通过 <Input ref={el => this.inputElement = el}/> 来获取 DOM 或者组件实例
  • 可以在组件间传递回调形式的 refs，实现 Refs 转发。
  • 注意：回调 Refs 是以内联函数的方式定义的，在更新过程中它会被执行两次。 因为在每次渲染时会创建一个新的函数实例，所以 React 清空旧的 ref 并且设置新的。
    • 第一次传入参数 null
    • 第二次会传入参数 DOM 元素

🍒React.forwardRef() - Refs 转发

ref 转发是一个可选特性，其允许某些组件接收 ref，并将其向下传递给子组件。

组件自身调用React.forwardRef((props,ref) => {...})，将 ref 向下转发到指定的子组件上。

  • 参数 ref 只在使用 React.forwardRef 定义组件时存在。常规函数和 class 组件不接收 ref 参数，且 props 中也不存在 ref。
  • Ref 转发不仅限于 DOM 组件，你也可以转发 refs 到 class 组件实例中。
