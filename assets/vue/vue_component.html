# data函数

data必须是一个函数，因为组件是复用的，需要做到作用域隔离。

# 渲染顺序

组件渲染顺序是由内而外的，子组件比父组件先渲染。

子组件的Mounted事件触发时，父组件的Mounted还未开始。所以监听要放在父组件的Created事件里。

# 组件注册

  1. 全局注册
  2. 局部注册
  3. 基础组件的全局自动化注册 nodejs 的 require.context() 功能。(svg icons等多处自动化批量处理都会用到这个方法）
import Vue from 'vue'
import upperFirst from 'lodash/upperFirst'
import camelCase from 'lodash/camelCase'

const requireComponent = require.context(
    // 其组件目录的相对路径
    '.',
    // 是否查询其子目录
    false,
    // 匹配基础组件文件名的正则表达式
    /[a-z]\w\.vue$/
)

requireComponent.keys().forEach(fileName => {
    const componentConfig = requireComponent(fileName)

    const componentName = upperFirst(
        camelCase(fileName.split('/').pop().replace(/\.\w+$/, ''))
    )

    // 如果这个组件是 export default 导出的，则优先使用.default
    Vue.component(componentName, componentConfig.default || componentConfig)
});

# Prop

Prop 是你可以在组件上注册的一些自定义 attribute 。

# 传递

  1. 静态prop
  2. 动态Prop通过v-bind来绑定动态值

注： 传入一个对象的所有属性,使用不带参数的 v-bind 实现: v-bind=prop-name

# 校验

  1. type: 类型检查

    • null 和 undefined 会通过所有类型检查
    • 类型可以为原生构造函数：String、Date、Symbol...
    • 多个类型可以为数组：[String, Number]
    • 类型也可以是自定义构造函数,通过 instanceof 判断

  2. required: 必填检查

  3. default: 默认值必填

    • 对象或数组默认值必须从一个工厂函数获取

  4. validator: 自定义验证函数

 validator: function (value) {
     // 这个值必须匹配下列字符串中的一个
     return ['success', 'warning', 'danger'].indexOf(value) !== -1
}

# 单向数据流

所有的prop是从父到子的 单向下行绑定 。不应该在子组件内部修改prop。

  1. 这个 prop 用来传递一个初始值；这个子组件接下来希望将其作为一个本地的 prop 数据来使用。

    定义一个本地的 data 属性并将这个 prop 用作其初始值

  2. 这个 prop 以一种原始的值传入且需要进行转换。

    定义一个这个prop的计算属性

注： 对象和数组是通过引用传入的，组件中改变这个对象或数组本身将会影响到父组件的状态。

# Prop的双向绑定：.sync修饰符

真正的双向绑定会带来维护上的问题。.sync修饰符是 Prop 与update:prop-name事件的语法糖:

<custom-component
  v-bind:title="doc.title"
  v-on:update:title="doc.title = $event"
></custom-component>

// 简写
<custom-component
  v-bind:title.sync="doc.title"
></custom-component>

# 非 props 的 attribute

组件可以接受任意的 attribute，而这些 attribute 会被添加到这个组件的根元素上。

# 1. 替换/合并
  1. 大多数属性，外部的属性会替换组件属性值。
  2. class和style会智能合并。
# 2. inheritAttrs: false

不希望 attribute 被默认添加到根元素上。可是设置该属性为false。

  • 该属性的设置不会影响 class 和 style。也就是说 class 和 style 永远都会默认在根元素上。
  • 希望自己手动设置的其它属性时候，使用$attrs可以获取到所有的attribute。

# 自定义事件

# 命名规范

由于 v-on 事件监听器在DOM模版中会被自动转为全小写（html大小写不明感），但事件名必须完全匹配。所以始终使用 kebab-case 的命名规范。

# v-model

是 prop 名为 value 和事件名为 input 的语法糖：

<component :value="value" @input="handleInput"></component>
1

model为了避免原生组件 value 用于不同目的的冲突，可以手动指定 prop 和 事件名。

注： prop 必须在 props 中申明。

# 监听原生事件：.native 修饰符

注： 当你监听类似 input 这种特定元素的原生事件 focus 时，会默默失败，因为组件根元素实际可能是个 label 元素。

解决方案： $listeners属性，包含了所有作用在这个组件上的事件监听器。配合 v-on="$listeners"将所有的事件监听器指向这个组件的某个特定的子元素

# 程序化的事件侦听

场景： 事件在代码里注册，但注销却在beforeDestory的生命周期里，业务逻辑是分散的。而程序化的事件侦听可以将业务逻辑写在一起。便于我们理解和清理。

# $on(event, callback)

监听当前实例上的自定义事件。事件可以是由 vm.$event 触发的事件。

# $once(event, callback)

监听一个自定义事件，但只能触发一次。一旦触发，监听就会被移除。

# $off([event, callback])

移除自定义事件监听器

  • $off() 没有提供参数，则移除所有监听器
  • $off(event_name) 只提供事件，则移除该事件的所有的监听器
  • $off(event_name, callback_name) 如果提供了事件和回调，则只移除这个回调的监听器

# $emit(eventName, [...args])

程序化触发事件。

this.$once('hook:beforeDestory',function(){
    func.destory()
})

# 透明组件Input

  1. input输入框的change 和input事件有何区别？

input 是输入中不断触发。change 只有在 blur 后触发

  1. composition 事件

中文输入过程中，会有拼写，如果不想拼写时触发更新，可以监听 compositionStart。

import Vue from 'vue'

Vue.component('base-input', {
    inheritAttrs: false,
    props: ['label', 'value'],
    computed: {
        inputListeners: function () {
            var vm = this;
            return Object.assign(
                {},
                // 父级添加的所有监听器
                this.$listeners,
                // 这是自定义的监听器，会覆盖一些父级监听事件
                {
                    // v-model 的工作
                    input: function (event) {
                        vm.$emit('input', event.target.value)
                    }
                })
        }
    },
    template: `
    <label>
        {{label}}
        <input v-bind="$attrs"  v-bind:value="value" v-on="inputListeners"></input>
    </label>

    `
})
