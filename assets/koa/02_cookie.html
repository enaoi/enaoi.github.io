# Cookie 操作

Koa2的cookie模块内嵌了cookies库

签名cookie： cookie存储在客户端，及时设置了httpOnly，各种代理也很容易获取到cookie信息。所以签名cookie的生成和加密在服务端进行，可以提高cookie的安全性。然后cookie是明文传输的，安全性依旧不高。所以cookie中不应该存储敏感信息，而只是保存 session_id。

# 读cookie

ctx.cookies.get('name')
1

签名cookie的获取，如果没有改变，则返回cookie的原始值，如果改变了，则返回undefined

ctx.cookies.get('cid',{signed: true})
1

# 设置cookie

  • maxAge 一个数字表示从 Date.now() 得到的毫秒数。 -1或是0时，则表示删除该cookie
  • signed cookie 签名值，为true时必须设置 app.keys(签名的cookie 密钥数组).cookie中新增 name.sig
  • expires cookie 过期的 Date
  • path cookie 路径, 默认是'/'
  • domain cookie 域名
  • secure 安全 cookie
  • httpOnly 服务器可访问 cookie, 默认是 true
  • overwrite 一个布尔值，表示是否覆盖以前设置的同名的 cookie (默认是 false). 如果是 true, 在同一个请求中设置相同名称的所有 Cookie（不管路径或域）是否在设置此Cookie 时从 Set-Cookie 消息头中过滤掉。
app.keys = ['awesomekey']

ctx.cookies.set('cid', 'helloworld',{
      domain:'localhost',
      path:'/cookie',
      maxAge: 10*60*1000,
      expires: new Date('2020-5-21'),
      httpOnly:false,
      overwrite: false,
      signed: true
    })
