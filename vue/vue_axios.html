# 重复请求处理

相同请求：method + url + params + data 都相等

当一个请求发出时，而相同的前一个请求还在pending状态，有多种选择：

  1. 取消旧的请求，以最新请求为准。
  2. 取消最新请求，等待老的请求返回结果。
  3. 多个相同请求pending，一旦返回一个请求，取消其它相同的pending请求。
  4. 将新的请求延迟....

以下为第一种情况的实现，始终以最新的请求为准：

  1. 在 axios 请求拦截器中保存请求信息，在响应拦截器中删除缓存。

    // @/api/http.ts
import Axios, { AxiosRequestConfig, AxiosResponse } from "axios";
import { Notification } from "element-ui";
import { RequestModel } from "@/store/request.vuex";

const addPending = (config: AxiosRequestConfig) => {
    const url = [
        config.method,
        config.url,
        JSON.stringify(config.params),
        JSON.stringify(config.data)
    ].join("&");
    const source = Axios.CancelToken.source();
    config.cancelToken = source.token;
    RequestModel.addCancelToken({ url, source });
};

const removePending = (response: AxiosResponse) => {
    const url = [
        response.request.method,
        response.request.url,
        JSON.stringify(response.request.params),
        JSON.stringify(response.request.data)
    ].join("&");
    RequestModel.removeCancelToken(url);
};

const service = Axios.create();

service.interceptors.request.use( config => {
        addPending(config);
        return config;
    }, error => {
        Notification.error({ title: "错误提示", message: "请求发生错误！" });
        return Promise.reject(error);
    }
);

service.interceptors.response.use( response => {
    removePending(response);
    if (response.status &&
    response.status === 200 &&
    response.data.status === 0 ) {
        return response.data.data;
    } else {
        Notification.error({ title: "错误提示", message: response.data.msg });
        return Promise.reject(response);
    }
}, error => {
    // cancel 的请求不需要提示信息
    if (!Axios.isCancel(error)) {
        const { status, responseURL } = error.request;
        Notification.error({
            title: "错误",
            message: `${responseURL}响应发生${status}错误！`
        });
    }
    return Promise.reject(error.response);
});

export default service;

  2. 在vuex中判断当前请求是否包含url，如果有，则cancel旧的请求，并且重新set 新的请求

    // @/store/request.vuex
import { VuexModule,getModule,Module,Mutation,Action } from "vuex-module-decorators";
import { CancelTokenSource } from "axios";
import store from "@/store";

export interface RequestState {
    cancelTokens: Map<string, CancelTokenSource>;
}

export interface TokenItem {
    url: string;
    source: CancelTokenSource;
}

@Module({ dynamic: true, store, name: "request", namespaced: true })
class Request extends VuexModule implements RequestState {
    // cancelTokens 是一个Map数据格式，key为url
    public cancelTokens: Map<string, CancelTokenSource> = new Map();

    @Mutation
    private ADD_CANCEL_TOKEN({ url, source }: TokenItem) {
        if (this.cancelTokens.has(url)) {
            this.cancelTokens.get(url)?.cancel();
        }
        this.cancelTokens.set(url, source);
    }

    @Mutation
    private REMOVE_CANCEL_TOKEN(url: string) {
        this.cancelTokens.get(url)?.cancel();
        this.cancelTokens.delete(url);
    }

    @Mutation
    private CLEAR_CANCEL_TOKEN() {
        this.cancelTokens = new Map();
    }

    @Action
    public CANCEL_PENDING_REQUESTS() {
        for (const [, request] of this.cancelTokens) {
        if (request.cancel) {
            request.cancel();
        }
        }
        this.CLEAR_CANCEL_TOKEN();
    }

    @Action
    public addCancelToken(config: TokenItem) {
        this.ADD_CANCEL_TOKEN(config);
    }

    @Action
    public removeCancelToken(url: string) {
        this.REMOVE_CANCEL_TOKEN(url);
    }
}

export const RequestModel = getModule(Request);

# 路由切换

路由切换时，该页面还在pending状态的请求，可以全部cancel。主要涉及三个模块：

  1. aixos 的request 拦截器内存储所有的发送请求
  2. vue-router 内拦截器，每次路由切换时，发起取消所有pending状态的请求操作
  3. vuex 内存储所有的请求记录，在 vue-router 发起请求时，触发还在pending状态的请求的abort方法，并清除请求记录。

主要实现如下：

  1. vuex 设置同上 @/store/request.vuex

  2. axios 拦截器设置同上

  3. vue-router 中通过路由守卫来在每次路由变化时，dispatch action。

    // @/router/index.ts
router.beforeEach((to, from, next) => {
  store.dispatch('app/CANCEL_PENDING_REQUESTS');
  next();
})
export default router;
