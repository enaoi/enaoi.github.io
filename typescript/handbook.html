# 基础类型

基础类型：

类型 表达式 说明
布尔 let isDone: boolean = false
数字 let binaryLiteral: number = 0b1010 所有数字都是浮点数，除了支持十进制和十六进制，还支持ECMAScript2015中引入的二进制和八进制
字符串 let name: string = "bob"
数组 let list: number[] = [1,2,3]、let list: Array<number> = [1,2,3] 只读数组: ReadonlyArray<T> ,只读数组也不能赋值给普通变量，只能通过类型断言为普通数组类解决
元组Tuple let x: [string, number] 已知长度和类型的特殊数组申明，越界元素会使用联合类型代替
枚举 enum Color {Red = 1, Green, Boolean} let c:Color = Color.red let b:Color = Color[2] 默认情况，下标从0 开始。可以手动制定成员的数值。
Any let notSure: any = 4、let list: any[] = [1, true, 'string'] 允许你在编译时可选择地包含或移除类型检查
Void function warnUser():void {}
Null 和 Undefined let u: undefined = undefined; let n: null = null; 默认情况下null和undefined是所有类型的子类型。 就是说你可以把 null和undefined赋值给number类型的变量。
Never function error(message：string): never { throw new Error(message)} 总会抛出异常和不会有返回值的函数； 被永不为真的类型保护所约束的变量。
Object declare function create(o: object|null): void

类型断言：

JSX中只支持as语法！

let someValue: any = 'this is a string.'
// 1. 尖括号
let strLength: number = (<string>someValue).length;
// 2. as 语法
let strLength: number = (someValue as string).length;

只要传入的对象满足必要条件，那么它就是被允许的。类型检查器不会检查属性的顺序，只要相应的属性存在并且类型也是对的就可以。多余的属性不会检查。

WARNING

对象字面量会被特殊对待，会经过额外属性检查。跳过检查有3中方法

  1. 使用类型断言: printLabel({ size: 10, label: "size 10" } as LabelValue)
  2. 添加一个字符串索引签名。
  3. 把对象字面量赋值给一个变量。就不会引起额外属性检查...

# 数组和类数组

typescript 支持两种索引签名：string、number。

注意

JavaScript会将 number 转化为 string 类型，然后再去索引对象。 所以数字索引必须是字符串索引返回类型的子类型。


 






interface StringArray {
  [index: number]: string;
}

let myArray: StringArray = ['Fetch', 'Fred']

myArray[1] === myArray['1'] // true

# 接口继承类

当接口继承类的时候，它会继承类的成员但不包括其实现。也会继承到类的private和protected成员。

意味着当你创建一个接口继承了一个拥有私有或受保护成员的类时，这个接口类型只能被这个类或它的子类所 implements。

class Control {
  private state: any;
}

interface SelectableControl extends Control {
  select(): void;
}

class Button extends Control implements SelectableControl {
  select() { }
}

// ERROR: Other 并不是 Control,缺少私有属性 state
// Other 缺少私有属性 private
class Other implements SelectableControl {
  select() {}
}
