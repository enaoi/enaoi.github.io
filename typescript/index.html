# TypeScript

🍒  参考文档

  • TypeScript handbook、中文
  • TypeScript Deep Drive、中文

🍒  为什么使用 TypeScript

  • 为 javascript 提供可选的类型系统
    • 类型可以让你在重构时更加敏捷，编译器更容易发现错误
    • 类型是最好的文档
  • 它为当前的引擎，提供未来 JavaScript 版本中已规划的功能
    • 类型错误并不阻止 JavaScript 编译

🍒  运行 TypeScript 文件

  1. npm install typescript -D
  2. npx ts-node path/to/tsfile

# 基础类型

  • boolean | string | number
  • 数组
    • number[]
    • Array<number>
    • 只读数组: ReadonlyArray<T>。只读数组也不能赋值给普通变量，只能通过类型断言为普通数组类解决。
  • 元组 Tuple：已知长度和类型的特殊数组申明。
    • let x: [string, number]
    • 越界元素会使用联合类型
  • 枚举：JavaScript 类型的补充。
    • enum Color {Red = 1, Green, Boolean}
    • 默认情况，下标从 0 开始。可以手动制定成员的数值。
    • 索引：let c:Color = Color.red | let b:Color = Color[2]
  • any：任何类型。
  • void：没有任何类型，一般都是函数没有返回值。
  • null 和 undefined 类型是所有类型的子集。
    • 指定了--strictNullChecks标记,null 和 undefined 只能赋值给 void 和它们各自。
  • never：永不存在的值的类型。
    • never 类型是任何类型的子类型，也可以赋值给任何类型
    • 没有类型是 never 的子类型或可以赋值给 never 类型（除了 never 本身之外）。即使 any 也不可以赋值给 never。
  • object： 非原始类型

🍒  类型断言

  • 尖括号：<string>(someValue)
  • as 语法：(someValue as string)
    • TypeScript 的 JSX 中只支持as语法

# 接口 Interface

🍒  普通对象

  • 可选属性：color?:string
  • 只读属性：readonly x:number
  • 额外的属性检查：对象字面量会被特殊对待，存在任何不包含的属性时报错。解决方案：
    • 使用类型断言: printLabel({ size: 10, label: "size 10" } as LabelValue)
    • 添加一个字符串索引签名。[propName: string]: any
    • 把对象字面量赋值给一个变量。

🍒  函数


 






interface SearchFunc {
  (source: string, subString: string): boolean;
}
let mySearch: SearchFunc;
mySearch = function(src, sub) {
  return src.search(sub) > -1;
};

🍒  可索引的类型 可索引类型具有一个索引签名。typescript 支持两种索引签名类型

  • number
    • 数字索引的返回值必须是字符串索引返回值类型的子类型。 因为用 number 来索引时，JavaScript 会将它转换成 string 然后再去索引对象。 100 等同于"100"，因此两者需要保持一致。
  • string 将索引签名设置为只读，可以防止给索引赋值
interface ReadonlyStringArray {
  readonly [index: number]: string;
}
let arr:ReadonlyStringArray = ["Fiona"]
arr[2] = "Mallory"; // error!

🍒  类类型 接口描述了类的公共部分，不包含私有部分

interface ClockInterface {
  currentTime: Date;
  setTime(d: Date);
}

class Clock implements ClockInterface {
  currentTime: Date;
  setTime(d: Date) {
    this.currentTime = d;
  }
  constructor(h: number, m: number) {}
}

🍒  继承接口 和类一样，接口也可以相互继承。

interface Shape {
  color: string;
}

interface PenStroke {
  penWidth: number;
}

// 单个继承
interface Square extends Shape {
  sideLength: number;
}
// 多个继承
interface Square extends Shape, PenStroke {
  sideLength: number;
}

🍒  混合类型 一个对象可以同时做为函数和对象使用，并带有额外的属性。

interface Counter {
  (start: number): string;
  interval: number;
  reset(): void;
}

function getCounter(): Counter {
  let counter = <Counter>function(start: number) {};
  counter.interval = 123;
  counter.reset = function() {};
  return counter;
}

let c = getCounter();
c(10);
c.reset();
c.interval = 5.0;

🍒  接口继承类 当接口继承类的时候，它会继承类的成员但不包括其实现。也会继承到类的 private 和 protected 成员。这个接口类型只能被这个类或它的子类所 implements

class Control {
  private state: any;
}
// 接口继承类
interface SelectableControl extends Control {
  select(): void;
}

// 子类实现接口
class Button extends Control implements SelectableControl {
  select() {}
}

// 错误：“Image”类型缺少“state”属性。
class Image implements SelectableControl {
  select() {}
}

# 枚举

🍒  数字枚举

  • 数字枚举默认从 0 开始
  • 具有自增长行为
  • 数字枚举如果前面的成员是非数字常量或者其它常量，需要手动初始化。

🍒  字符串枚举

  • 没有自增长行为
  • 每个成员都必须用字符串字面量

🍒  异构枚举

  • 枚举可以混合字符串和数字成员，但不建议。

🍒  const 枚举

  • 不同于常规的枚举，它们在编译阶段会被删除
  • 常量枚举不允许包含计算成员
const enum Enum {
  A = 1,
  B = A * 2,
}

🍒  外部枚举

  • 在正常的枚举里，没有初始化方法的成员被当成常数成员
  • 没有初始化方法时被当做需要经过计算的
declare enum Enum {
  A = 1,
  B,
  C = 2,
}
