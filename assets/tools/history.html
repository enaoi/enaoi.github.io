# 开始

history 提供了三种方法，来适应不同的环境：

  • createBrowserHistory: 所有浏览器
  • createMemoryHistory: 在浏览器中使用URL的哈希部分来标记路由，避免了加载页面时向服务器发送请求
  • createMemoryHistory: 内存history，在非浏览器环境中使用，比如react native 或者测试环境
import { createBrowserHistory } from 'history'
let history = createBrowserHistory();

const { location, action } = history;

// 监听当前location的变化
let unlisten = history.listen(({ location, action }) => {
  console.log(action, location.pathname, location.state);
  console.log(`The last navigation action was ${action}`);
});

// 调用listen返回的函数，取消监听
unlisten()

// push 一个新的实体进入 history 栈
history.push('/home', { some: 'state' });

// replace
history.replace('/logged-in');

// 使用 back/forward
history.back()

# Properties

每一个 history 实体都有以下两个属性

  • history.location: window.location的子集
    • location.pathname
    • location.search
    • location.hash
    • location.state: 传递的数据，一般为null
    • location.key
  • history.action: 包含三种类型
    • Action.Push
    • Action.Replace
    • Action.Pop: 进入已存在栈中的地址

memory history 提供了一个 history.index 来标记你当前location在history栈中的index

# 导航

history实例通过编程方式修改当前location，有以下几个接口

  • history.push(to: To, state?: State)
  • history.replace(to: To, state?: State)
  • history.go(delta: number)
  • history.back()
  • history.forward()
history.push('/home');

// Push a new entry onto the history stack with a query string
// and some state. Location state does not appear in the URL.
history.push('/home?the=query', { some: 'state' });

// If you prefer, use a location-like object to specify the URL.
// This is equivalent to the example above.
history.push({
  pathname: '/home',
  search: '?the=query'
}, {
  some: state
});

// Go back to the previous history entry. The following
// two lines are synonymous.
history.go(-1);
history.back();
