# 作用域

变量提升：

JavaScript 引擎的工作方式是，先解析代码，获取所有被声明的变量，然后再一行一行地运行。这造成的结果，就是所有的变量的声明语句，都会被提升到代码的头部，这就叫做变量提升（hoisting）

合法标识符：

  1. 第一个字符，可以是任意 Unicode 字母（包括英文字母和其他语言的字母），以及美元符号（$）和下划线（_）。
  2. 第二个字符及后面的字符，除了 Unicode 字母、美元符号和下划线，还可以用数字 0-9。

# this

this 是 javascript 的关键字，被自动定义在所有函数作用域中。

👉为什么使用 this

this 提供了一种更优雅的方式来隐式“传递”一个对象引用，因此可以将 API 设计得更加简洁并且易于复用，不需要显式的传入上下文对象。

👉this 指向的误解

误解 1： this 指向当前函数对象本身

function foo(number) {
  this.count++;
}

for (let i = 0; i < 6; i++) {
  foo(i);
}

foo.count; // => undefined
count; // => NaN

this 并非是指向函数对象本身。但是全局变量中多了一个 count，它的值为 NaN。

如果我们要将一个函数的 this 指向函数对象本身，需要使用foo.call( foo, i )

误解 2： this 指向当前函数的作用域

这个问题有点复杂，因为有的时候，它是正确的，有的时候它是错的。

👉全局环境中的 this

  • 在非严格模式下，this 指向 window
  • 在严格模式下，this 指向 undefined

👉this 的指向

  • this 是在运行时进行绑定的，并不是在编写时绑定，它的上下文取决于函数调用时的各种条件。
  • this 的绑定和函数申明的位置没有任何关系，只取决于函数的调用方式。
  1. 默认绑定：简单的函数调用
  2. 隐式绑定：链式调用，且只有最近的一个包含函数的对象会影响调用位置
  3. 隐式丢失：被隐式绑定的函数会丢失绑定对象，而应用默认绑定。常见的是回调参数传入函数和函数别名赋值。
  4. 显示绑定：使用call()、apply()、bind()方法

call、apply、bind 有什么区别 ？

这些方法都是用来改变相关函数 this 的指向，但是 call 和 apply 是直接进行相关函数调用的，bind 不会执行相关函数，而是返回一个新的函数，这个新的函数已经自动绑定了新的 this 指向。 call 和 apply 的主要区别就是体现在参数上。

👉构造函数中的 this 指向

new 操作符的执行过程如下：

  1. 创建一个新的空对象。
  2. 新对象的__proto__指向构造函数的 prototype 属性。
  3. 将 1 创建的新对像做为 this 的上下文,并且执行。
  4. 如果执行结果是个对象，且不为 null，那么返回这个返回值，如果不是则返回这个新建的对象。

根据上面的描述可以写出下面实现的方案

function ObjectFactory() {
  let newObject = {};
  const fn = [].shift.apply(arguments);
  newObject.__proto__ = fn.prototype;
  var result = fn.apply(newObject, arguments);
  if (typeof result === "object" && result !== null) {
    return result;
  }
  return newObject;
}

由构造函数 new 操作的原理可知，如果构造函数的执行结果是一个不为 null 的对象，那么它的 this 就是这个执行结果，否则（返回数字等或者没有返回值）就是实例。

👉箭头函数中的 this 指向

在箭头函数中，this 指向是由外层（函数或者全局）作用域来决定的。

🎈 箭头函数体内的 this 对象，就是定义时所在的对象，而不是使用时所在的对象。

👉this 的优先级

  1. new 绑定的优先级高于显式绑定。
  2. 普通函数内的 this 是可变的，但是箭头函数内，它是固定的。在定义时所在的对象，而不是调用时的。
var a = 123;
function foo() {
  return () => {
    console.log(this.a);
  };
}

const obj1 = {
  a: 2,
};

const obj2 = {
  a: 3,
};

const bar = foo.call(obj1);
console.log(bar.call(obj2)); // 2

// foo 函数改写为全箭头函数后
const foo = () => () => {
  console.log(this.a);
};

console.log(bar.call(obj2)); // 123

// 将 var 改为 const。const 申明的变量不会挂在到 window 全局对象上。
const a = 123;
console.log(bar.call(obj2)); // undefined

👉实现一个 bind 函数

// TODO:
// 1. bund函数如果作为构造函数搭配new返回的话，this就会被忽略
// 2.函数又一个length属性，下面的方法没有对length的重写和还原
Function.prototype.bind = function(context) {
  var _self = this;
  var args = Array.prototype.slice.apply(arguments, 1);
  return function funbind() {
    var innerArgs = Array.prototype.slice.call(arguments);
    var finalArgs = args.concat(innerArgs);
    return _self.apply(context, finalArgs);
  };
};

// 测试部分：
function foo(a) {
  this.a = a;
}

const obj1 = {};

var bar = foo.bind(obj1);

// 普通 bind 的调用
bar(2);
console.log(obj1.a);

// new 调用
var baz = new bar(3);
console.log(baz.a);
console.log(obj1.a);
// 正常情况下，new的优先级会高于bind。
