# Fetch

官方文档:

  • MDN Fetch
  • fetch 规范

参考文档:

  • 进阶指南
  • fetch ApI

XHR 的缺点:

  1. MLHttpRequest 是一个设计粗糙的 API，不符合关注分离（Separation of Concerns）的原则，配置和调用方式非常混乱
  2. 基于事件的异步模型写起来也没有现代的 Promise，async/await 友好

Fetch 的优点:

  1. 语法简洁，更加语义化
  2. 基于标准 Promise 实现，支持 async/await
  3. 同构方便，使用 isomorphic-fetch
  4. 提供更丰富的API（request、response、body、headers）

兼容问题:

  1. chrome42~45,fetch对中文支持有问题
  2. 原生fetch支持率不高，IE中普遍不支持

polyfill:

  • es5 的 polyfill: es5-shim, es5-sham
  • Promise 的 polyfill: es6-promise
  • fetch 的 polyfill: fetch-ie8
  • 可选：IE8下的xhr支持 CORS 跨域，虽然提供 XDomainRequest， 但不支持cookie，所以如果接口需要权限验证，使用 jsonp: fetch-jsonp

Fetch polyfill 的基本原理是探测是否存在 window.fetch 方法，如果没有则用 XHR 实现。这也是 github/fetch 的做法，但是有些浏览器（Chrome 45）原生支持 Fetch，但响应中有中文时会乱码，所以使用 fetch-detector 和 fetch-ie8 只在浏览器稳定支持 Fetch 情况下才使用原生 Fetch。

常见问题:

  • 默认情况下，fetch 不会从服务端发送或接收任何 cookies, 如果站点依赖于用户 session，则会导致未经认证的请求。需要设置 fetch(url, {credentials: 'include'})
  • 服务器返回 400，500 错误码时并不会 reject（但是会将 resolve 的返回值的 ok 属性设置为 false ）。当网络故障时或请求被阻止时，fetch 才会被 reject。
  • 不支持 abort, 不支持超时控制、造成了流量浪费

abort解决方案: 使用 Promise.race() 任何一个 Promise 被 resolve 或 reject 后，外部的Promise 就会以相同的值被 resolve 或 reject。

    var _fetch = (function (fetch) {
      return function (url, options) {
        var abort = null;
        var abort_promise = new Promise((resolve, reject) => {
          abort = () => {
            reject('abort.');
            console.info('abort done.');
          };
        });
        var promise = Promise.race([
          fetch(url, options),
          abort_promise
        ]);
        promise.abort = abort;
        return promise;
      };
    })(fetch);

timeout 机制: 依旧可以使用 Promise.race()

   function timer(t) {
      return new Promise(resolve => setTimeout(resolve, t))
        .then(function (res) {
          console.log('timeout');
        });
    }
    var p = fetch('https://www.baidu.com', { mode: 'no-cors' });
    Promise.race([p, timer(1000)]);

xhr 版本的timeout

var _fetch = (function (fetch) {
      return function (url, options) {
        var abort = null,
          timeout = 0;
        var abort_promise = new Promise((resolve, reject) => {
          abort = () => {
            reject('timeout.');
            console.info('abort done.');
          };
        });
        var promise = Promise.race([
          fetch(url, options),
          abort_promise
        ]);
        promise.abort = abort;
        Object.defineProperty(promise, 'timeout', {
          set: function (ts) {
            if ((ts = +ts)) {
              timeout = ts;
              setTimeout(abort, ts);
            }
          },
          get: function () {
            return timeout;
          }
        });
        return promise;
      };
    })(fetch);

progress:

function consume(reader) {
      var total = 0
      return new Promise((resolve, reject) => {
        function pump() {
          reader.read().then(({ done, value }) => {
            if (done) {
              resolve();
              return;
            }
            total += value.byteLength;
            console.log(`received ${value.byteLength} bytes (${total} bytes in total)`);
            pump();
          }).catch(reject)
        }
        pump();
      });
    }

    fetch('http://localhost:10101/notification/', { mode: 'no-cors' })
      .then(res => consume(res.body.getReader()))
      .then(() => console.log("consumed the entire body without keeping the whole thing in memory!"))
      .catch(e => console.log("something went wrong: " + e));

options:

  1. method： GET、POST
  2. headers： Header 对象实例
  3. credentials
    • include: 跨域携带cookies
    • same-origin: 同域cookies
    • omit: 不携带cookies
  4. body
    • JSON
    • FormData
    • Bolb/BufferSource
  5. mode
    • cors：表示同域和带有CORS响应头的跨域下可以
    • same-origin : 必须同源，绝对禁止跨域
    • no-cors: 对外的请求可以发送，外域的服务器无论设不设置允许跨域都会接受请求并处理请求。但是浏览器不接受响应。即使外域返回了内容，浏览器也当做没接到。此时响应类型为 “opaque”.
  6. cache
    • default：表示 fetch 请求之前将检查下 http 的缓存
    • no-store：表示 fetch 请求将完全忽略 http 缓存的存在. 这意味着请求之前将不再检查下 http 的缓存, 拿到响应后, 它也不会更新 http 缓存
    • reload ：表示 fetch 请求之前将忽略http缓存的存在, 但是请求拿到响应后, 它将主动更新 http 缓存
    • no-cache：如果存在缓存, 那么 fetch 将发送一个条件查询 request 和一个正常的 request , 拿到响应后, 它会更新 http 缓存
    • force-cache：表示 fetch 请求不顾一切的依赖缓存, 即使缓存过期了, 它依然从缓存中读取. 除非没有任何缓存, 那么它将发送一个正常的request
    • only-if-cache：表示fetch请求不顾一切的依赖缓存, 即使缓存过期了, 它依然从缓存中读取. 如果没有缓存, 它将抛出网络错误(该设置只在mode为”same-origin”时有效)
  7. redirect
  8. referrer

如果fetch请求的header里包含 If-Modified-Since, If-None-Match, If-Unmodified-Since, If-Match, 或者 If-Range 之一, 且cache的值为 default , 那么fetch将自动把 cache的值设置为 "no-store" .

Headers 对象:

  1. 添加：append(name,value)：给现有的header添加一个值, 或者添加一个未存在的header并赋值
  2. 检索：has(name)：boolean
  3. 获取：get(name)
  4. 删除：delete(name)

Response对象

属性:

  • ok: Boolean 类型，当 HTTP status 范围在 [200-299] 是为 true。
  • type: ['basic','cors','opaque']
  • url
  • redirected
  • status: http code
  • statusText
  • headers: Headers 对象
  • body: 可读流
  • bodyUsed: 布尔类型

方法:

  • response.text()
  • response.json()
  • response.formData()
  • response.blob()
  • response.arrayBuffer()
// 原生XHR
var xhr = new XMLHttpRequest();
xhr.open('GET', url);
xhr.responseType = 'json';
xhr.onload = function () {
  console.log(xhr.response);
};
xhr.onerror = function () {
  console.log("Oops, error");
};
xhr.send();

// fetch es6
fetch(url).then(response => response.json())
  .then(data => console.log(data))
  .catch(e => console.log("Oops, error", e))
  
// fetch es7 async/await
const getUrl = async (url) => {
  try {
    let response = await fetch(url);
    let data = response.json();
    console.log(data);
  } catch (e) {
    console.log("Oops, error", e);
  }
}
