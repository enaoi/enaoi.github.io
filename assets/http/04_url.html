# 统一资源定位符 URL

服务器资源名被称为统一资源标识符(Uniform Resource Identifier，URI)。URI 有两种形式，分别称为 URL 和 URN。统一资源定位符(URL)是资源标识符最常见的形式。

URL 描述了一台特定服务器上某资源的特定位置。 URL 也可以通过 HTTP 协议之外的其它协议访问资源：

  • email: mailto:president@whitehouse.gov
  • 文件传输协议FTP：ftp://ftp.lots-o-books.com/pub/complete-list.xls
  • 流视频服务器：rtsp://www.joes-hard.com:554/interview/cot_video
  • ...

# URL语法

通用URL格式

<protocol>://<username>:<password>@<host>:<port>/<path>?<query>#<fragment>

最常见 http 协议使用的 URL 分析图

URL中的 # 符号

URL中的 # 部分称为 Fragment URL。浏览器会查询网页中:

  • name属性值匹配的<a>标签：<a name="hashvalue"/>
  • 或id属性匹配的任意标签：<div id="hashvalue"/>

匹配后，浏览器会将该部分滚动到可视区域的顶部。

🍉 总结

  • 仅仅作用于浏览器，不包含在请求中：注意提交的表单内容不能包含#，否则会被截断
  • #后面的所有字符，都会被浏览器当做位置标志符。匹配锚点。
  • 浏览器可以通过window.location.hash来读写#。
  • 修改#不会导致页面重新加载，但是会改变浏览器的历史记录，可以通过上一页按钮回到原始的位置。
  • 修改#值会触发 HTML5 的window.onhashchange事件

# 相对URL

绝对URL包含访问资源所需的全部信息，相对URL是不完整的，需要相对与base URL 进行解析。

优点：保持资源的便携性。在搬运一组资源时，仍然保持链接的有效性。

# 字符要求

URL字符要求：

  • 可移植：可以通过任意网络协议安全传输。安全传输意味着不能丢失信息，某些协议，比如说简单邮件传输协议(SMTP)会剥去一些特殊字符，所以URL只会使用相对较小的、通用的字符。
  • 可读：不可见、不可打印的字符不能使用
  • 完整：想要使用除了安全字符外的二进制数据或字符时，需要使用编码，将不安全的字符转义为安全字符。

# 字符限制

有一些保留字符，有特殊含义。除了将其用于保留用途外，需要在URL中对其进行编码。

字符 保留/受限
% 保留为编码字符的转义标志
/ 保留为分隔路径段的定界符
. 保留在路径组件中使用
.. 保留在路径组件中使用
# 保留作为Fragment符
？ 保留作为查询字符串定界符
; 保留作为参数定界符
: 保留作为协议、用户/口令、主机/端口定界符
$,+ 保留
@&= 在某些方案的上下文中有特殊的含义，保留
{}|^~[]' 受限，由于各种Agent代理
<>" 不安全
0x00~0x1F,0x7F 受限，US-ASCII字符集不可打印区间
>0x7F 受限，US-ASCII字符集包含128个字符，占一个字节的7个位置，首位默认为0，其它的是属于扩展字符。并不统一和安全。

# 常见方案

常见的传输协议方案总结

方案 描述
http 超文本传输协议，没有用户名和密码之外，与通用的URL格式相符。默认端口：80
https 与http方案一致，唯一区别是使用了SSL，为http连接提供了端到端的加密机制。默认端口：443
mailto 指向E-mail地址。较特殊的格式mailto:joe@joes-hardware.com
ftp 文件传输协议。URL可以用来从ftp服务器上下载或者向其上传文件，并获取FTP服务器的目录结构内容的列表。基本格式：ftp://<user>:<password>@<host>:<port>/<path>;<params>
rtsp,rtspu 实时流传输协议，解析音/视频媒体资源的标识符。U表示使用UDP协议来获取资源。基本格式：rtsp://<user>:<password>@<host>:<port>/<path>
file 指定一台主机的文件共享系统。基本格式file://<host>/<path>
