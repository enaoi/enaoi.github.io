# context 具体方法和访问器

  1. ctx.req/res Node的对象
  2. ctx.request
    • ctx.header
    • ctx.headers
    • ctx.method
    • ctx.url
    • ctx.originalUrl
    • ctx.origin 包括 protocol, host
    • ctx.href 包括 protocol, host, query
    • ctx.path
    • ctx.query
    • ctx.queryString
    • ctx.host
    • ctx.fresh 检查请求缓存是否“新鲜”，也就是内容没有改变。此方法用于 If-None-Match / ETag, 和 If-Modified-Since 和 Last-Modified 之间的缓存协商。 在设置一个或多个这些响应头后应该引用它。
    • ctx.stale 与 request.fresh 相反
    • ctx.secure 通过 ctx.protocol == "https" 来检查请求是否通过 TLS 发出。
    • ctx.socket
    • ctx.protocol
    • ctx.ip
    • ctx.get(filed) 返回请求头，field不区分大小写
    • ctx.is(types) 检查传入请求是否包含 Content-Type 消息头字段， 并且包含任意的 mime type。 如果没有请求主体，返回 null。 如果没有内容类型，或者匹配失败，则返回 false。 反之则返回匹配的 content-type。 内容协商
    • ctx.accepts(types)
    • ctx.acceptsEncodings(types)
    • ctx.acceptsCharsets(charsets)
    • ctx.acceptsLanguages(langs)
  3. ctx.response
    • ctx.body
    • ctx.status
    • ctx.message
    • ctx.length
    • ctx.type
    • ctx.headerSend
    • ctx.redirect()
    • ctx.set()
    • ctx.append()
    • ctx.remove()
    • ctx.lastModified
    • ctx.etag
  4. ctx.state
  5. ctx.app
    1. ctx.app.emit
  6. ctx.cookies
    1. ctx.cookies.get(name,[options])
    2. ctx.cookies.set(name,value,[options] options 的参数
      • maxAge
      • signed
      • expires
      • path
      • domain
      • secure
      • httpOnly：true
      • overwrite
  7. ctx.throw(status,msg) 基于 http-errors
  8. ctx.assert(value,status,msg,properties)
