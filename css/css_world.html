# 📚 CSS 世界

# 基础概念

# 元素尺寸的相关概念

  1. 元素尺寸（元素偏移尺寸）= 元素 + padding + border
    • JQuery：$().width()、$().height()
    • DOM API: offsetWidth和offsetHeight
  2. 元素内部尺寸（元素可视尺寸）= 元素 + padding
    • JQuery：$().innerWidth()、$().innerHeight()
    • DOM API: clientWidth和clientHeight
  3. 元素外部尺寸 = 元素 + padding + border + margin - JQuery：$().outerWidth()、$().outerHeight()

注意

元素外部尺寸有可能为负数。现实中没有什么物体的尺寸是负数的，所以外部尺寸可以理解为“元素占据的空间尺寸”。

# Margin

# margin 和元素的内部尺寸

margin 对元素的可视尺寸的改变和 padding 几乎是互补的。

  1. 元素设定了 width 或者保持包裹性状态：margin 对可视尺寸无效。
  2. 元素是“充分利用可用区间”状态：margin 可以改变元素的内部尺寸。
<div class="father">
    <div class="son"></div>
</div>

<style>
    .father { width: 300px;}
    .son { margin: 0-20px; }
</style>

👆 son 元素的宽度尺寸通过负值设置变大为 340 像素，但垂直方向的高度没有任何变化。这是为什么？

  • 普通流体元素：CSS 默认的流方向是水平的，margin 只能改变元素的水平方向尺寸。
  • 具有拉伸特性的绝对定位元素：水平或者垂直方向都可以。

# float

🎈 浮动元素：float 值为非 none 的元素。

特性：

  • 包裹性
  • 块状化并格式化上下文
  • 破坏文档流
  • 没有任何 margin 合并

# clear

clear属性值制定一个元素是否必须移动到在它之前的浮动元素下面。

属性值：

  • none: 默认值
  • left: 左侧抗浮动
  • right: 右侧抗浮动
  • both: 两侧抗浮动 ✔️

🎁 要被清除的相关浮动元素指的是在相同块级格式化上下文中的前置浮动。

❓ 存在的问题：

  1. clear 属性只对块级元素才有效。所以借助伪元素清除浮动的时候需要将默认内联元素设置为table或者block。
  2. clear本质上只是让自身不和前面的浮动元素一行显示，并不是实质上的清除浮动。对自身后面的浮动元素也不做任何处理。

# BFC

定义：BFC（Block Formatting Context) 块格式化上下文

触发 BFC 的常见情况:

  • <html>根元素
  • 浮动元素
  • position != relative | static
  • overflow != visible 的块元素
  • display = inline-block|table-cell|table-caption
  • 弹性元素(display=flex|inline-flex元素的直接子元素)

各种属性解析：

  • float:left。⭐️⭐️⭐️ 浮动元素本身 BFC 化，然而浮动元素有破坏性和包裹性，失去了元素本身的流体自适应性。无法实现自动填满容器的自适应布局。 但兼容性良好。
  • position:absolute。⭐️ 脱离文档流严重。
  • overflow:hidden。⭐️⭐️⭐️⭐️ 块状元素的流体特性保存良好，且有 BFC 的独立区域特性。
  • dispaly:inline-block。⭐️ 行内块级元素具有包裹性，但在 IE6 和 IE7 中，依旧 block 水平，自适应容器可用宽度，且触发 BFC 特性。*zoom:1也是类似效果，适用于低级 IE 浏览器。
  • dispaly:table-cell。⭐️⭐️⭐️ 具有包裹性，但它包含一个特殊特性：宽度值设置的再大，师级宽度也不会超过容器宽度，可以解决自适应容器宽度的问题。兼容：IE8+。处理连续英文换行较为麻烦。
  • display:table-row。对 width 无感，无法自适应剩余容器空间。
  • display:table-caption。

适合自适应布局的属性：

  • overflow:auto/hidden。适用于 IE7+
  • display:inline-block。IE6+IE7
  • display:table-cell。IE8+

兼容 IE7+及以上版本浏览器适配的自适应解决方案：

  1. overflow 属性
.lbf-content {
  overflow: hidden;
}
  1. display: table-cell 和 display: inline-block;
.lbf-content {
  display: table-cell;
  width: 99999px;
  /*兼容IE7*/
  *display: inline-block;
  *width: auto;
}

# oveflow

  • overflow 的裁切边界是 border box 的内边缘
  • 经典不兼容问题：容器可滚动模式下，padding-bottom 在 IE 和 FireFox 内被忽略。应该避免设置滚动容器的 padding-bottom 属性

🍒  overflow-x和overflow-y

  • IE8+兼容
  • 除非两个属性同时设置为visible,否则设置的visible的样式会表现为auto。visible是默认值，很多时候auto也不必多余书写。
** {
  overflow-x: hidden;
  overflow-y: auto; /* 多余:默认值visible会自动当做auto来解析 */
}

🍒  滚动条

  • PC 端，默认滚动条来自<html>，而不是<body>。去除 PC 端默认滚动条 html{ overflow: hidden; }。
  • 窗体滚动高度获取：
    • PC 端： document.documentElement.scrollTop
    • 移动端： document.body.scrollTop
  • 滚动条会占用容器的可用宽度和高度。
    • PC 端一般滚动条所占的宽度是 17px
    • 移动端一般滚动条是悬浮模式
  • 滚动条可能会导致页面加载时，布局产生晃动。🗒  解决方案示例

🍒  overflow 和锚点定位

锚点定位行为触发条件：

  1. url 地址中的锚点与锚点元素 对应 并且 有交互行为 。
  2. 可 focus 的锚点处于 focus 状态。
  • <a href="javascript:">返回顶部</a>，避免 url 中出现#。
  • url 定位是让元素出现在浏览器窗体的上边缘
  • focus 定位是让元素在浏览器窗体范围内显示即可，不一定是上边缘

锚点定位的本质：通过改变容器滚动高度或者宽度来实现。

  • 容器滚动高度不一定是浏览器滚动高度，锚点定位可能是普通元素容器。
  • 锚点定位行为，按由内而外的顺序发生。
  • ✨overflow:hidden 的元素依然可以滚动。

# position:absolute

特性：

  • 块状化并格式化上下文：display 计算属性变为 block 或 table，产生 BFC。
  • 包裹性
  • 破坏正常文档流

总结：

  • absolute 和 float 同时存在时，float 无效。
  • absolute 的自适应最大宽大不是由父元素决定的，而是“包含块”。

🍒  absolute 的包含块

常见包含块：

  • 根元素：（可以看做是<html>）初始包含块
  • position = relative|static,包含块 = 其最近的块容器祖先盒的 content box 边界形成
  • position = fixed,包含块 = 初始包含块
  • position = fixed,包含块 = 最近的 position 不为 static 的祖先元素建立。
    • 如果该祖先元素是纯 inline 元素，则规则较为复杂：
      • 假设给内联元素的前后各生成一个宽度为 0 的内联盒子（inline box），则这两个内联盒子的 padding box 外面的包围盒就是内联元素的“包含块”
      • 如果该内联元素被跨行分割了，那么“包含块”是未定义的，也就是 CSS2.1 规范并没有明确定义，浏览器自行发挥。
    • 如果该祖先元素是块级元素：祖先元素的 padding box 边界形成
    • 如果没有符合条件的祖先元素，则包含块 = 初始包含块

🍒  具有相对特性的无依赖 absolute 绝对定位

  • 一个绝对定位元素，没有任何 top/bottom/left/right 属性设置，还在当前位置。
  • absolute 是非常独立的 css 属性，不依赖于任何其他 css 属性
  • "无依赖绝对定位" = "相对定位"

"无依赖绝对定位"的适用场景：

  1. 各类 badge 定位。new|hot 等。兼容 IE6+。🗒  示例
  2. 超越常规布局的排版。🔗  demo
  3. 下拉列表的定位。🔗  demo
  4. 占位符效果模拟：IE9 及以下浏览器不支持 placeholder 占位符效果

🍒  absolute 和 text-align

<style>
  p {
    width: 300px;
    height: 120px;
    text-align: center;
  }
  img {
    position: absolute;
  }
</style>
<p>
  <img src="path/to/img1" />
  <p></p>
</p>

text-align 会改变 absolute 元素的位置。是”幽灵空白点“和”无依赖绝对定位“共同的结果：

  • 由于<img>是内联水平，标签中存在“幽灵空白节点”，也是内联水平，受 text-align:center 影响而水平居中显示。
  • <img>设置了position:absolute; 表现为”无依赖绝对定位“。于是在“幽灵空白节点”后面定位显示。
