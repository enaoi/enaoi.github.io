# XMLHttpRequest

# 属性

  1. readyState | value |常量名 |说明 | | --- | --- | --- | | 0 | xhr.UNSENT | XMLHttpRequest 对象已创建或已被 abort() 方法重置 | | 1 | xhr.OPENED | open() 方法已调用，可以使用实例的setRequestHeader()方法设置请求头信息 | | 2 | xhr.HEADERS_RECEIVED |Send() 方法已调用,响应头部都已经接收到 | | 3 | xhr.LOADING | 响应体开始接收但未完成 | | 4 | xhr.DONE | HTTP 响应已完全接收 |

  2. responseType： 手动设置。确保浏览器和设置的值保持一致。不兼容的时候，服务器返回的数据变为null。

    • "" = "text"
    • "json"
    • "blob"
    • "document"
    • "arraybuffer"

  3. response：做为响应主体被返回，手动设置的responseType会自动转换response的格式。

  4. responseText：纯文本内容。手动设置的type和它不一致时会报错

  5. status：服务器返回的HTTP状态码。readyState<3 时,读取该属性会导致异常

  6. statusText：status=200 时是 "OK"，status=404 时是 "Not Found"。readyState<3 时,读取该属性会导致异常

  7. responseURL： url

  8. responseXML：返回从服务器接收到的 HTML 或 XML 文档对象。 有效的前提：

    1. 发送请求前，xhr.responseType = 'document'
    2. 如果回应的Content-Type 不等于 text/xml 或application/xml。必须用 xhr.overrideMimeType('text/xml')强制进行 XML 解析

  9. withCredentials： 是否允许跨域携带cookie

    • 服务端必须显示返回 Access-Control-Allow-Credentials: true 才能让属性生效

  10. timeout: 0 表示没有时间限制。时间单位为ms，表示多少毫秒后，请求为超时，自动终止。

  11. upload: 文件上传

# 事件句柄

  1. onreadystatechange：每次 readyState 属性改变的时候调用的事件句柄函数。当 readyState 为 3 时，它也可能调用多次。

  2. onload: 请求成功完成

  3. onerror：请求失败完成

  4. onloadend：请求完成：不管成功还是失败

  5. ontimeout

  6. onabort

  7. onloadstart: HTTP 请求发出

  8. onprogress: 正在发送和加载数据

# 实例方法

  1. open(method, url, async, username, password)：初始化 HTTP 请求参数。async 默认为true

    1. async:false：请求是同步的，后续对 send() 的调用将阻塞，直到响应完全接收。
    2. async:true：请求是异步的，且通常需要一个 onreadystatechange 事件句柄。

  2. setRequestHeader(name, value)： 必须在调用 open() 后，send() 之前，设置或添加一个 HTTP 请求头部。例如 xhr.setRequestHeader('Accept','application/json, text/plain, */*')

  3. getResponseHeader(name)：name不区分大小写

  4. getAllResponseHeaders()：axios中就是调用该方法，将 headers 做为 response 对象的参数返回。字符串类型，使用CRLF分隔（回车+换行）

  5. send(body)

    1. 请求发布，readyState 由 1 置为 2，并触发 onreadystatechange 事件句柄。
    2. 同步请求：这个方法会阻塞并不会返回，直到 readyState 为 4 并且服务器的响应被完全接收；异步请求：send() 立即返回，服务器响应将在一个后台线程中处理。
    3. 如果服务器响应带有一个 HTTP 重定向，send() 方法或后台线程自动遵从重定向。
    4. 当所有的 HTTP 响应头部已经接收，send() 或后台线程把 readyState 置为 3， 并触发 onreadystatechange 事件句柄。
    5. 如果响应较长，send() 或后台线程可能在状态 3 中触发 onreadystatechange 事件句柄：这可以作为一个下载进度指示器。
    6. 当响应完成，send() 或后台线程把 readyState 设置为 4，并最后一次触发事件句柄。

  6. abort()：取消当前响应，把 XMLHttpRequest 对象重置为 readyState 为 0 的状态,关闭连接并且结束任何未决的网络活动。
